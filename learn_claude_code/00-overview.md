# 0. はじめに

このドキュメントは、Claude Code を初めて触るジュニアエンジニア向けに、
「手を動かしながら確実に成果を出す」ための考え方と使い方を整理したものです。
ただし内容はチーム運用や設計の話まで踏み込むため、シニアにも学びがある構成にしています。

## このドキュメントのゴール
- Claude Code の全体像と「どこが価値の出しどころか」を理解する
- プロンプトとコンテキストの設計で再現性を上げる
- 機能（slash commands / skills / sub-agents / hooks / MCP）を実務で使える
- チームで運用できるレベルのルール設計ができる

## 進め方（推奨）
1. Quickstart で 1 回動かす
2. 「概念」→「プロンプト」→「コンテキスト」で土台を作る
3. 「各機能の使い方」で機能を把握する
4. 「応用」で実務に当てはめる
5. spec-workflow MCP 連携で“設計の型”を作る
6. ドキュメントベストプラクティスで運用を強化する

## 前提知識
- Git と基本的な CLI 操作
- 1 つ以上のプロジェクトでの実装経験

## 最初の練習（5分）
1. 今やっているタスクを 1 つ選ぶ
2. 目的・制約・完了条件を 1 行ずつ書く
3. Claude Code にそのまま投げる

## セルフチェック
- 目的・制約・完了条件が書けている
- 変更範囲が明確になっている
- 「わからない点は質問して」と伝えた
